# Instructions — Phase 12: Backend Specialist Agent

> Add this file to your IDE's custom instructions when building or extending the Backend Specialist Agent.

---

## Context

You are building the **AD Ports Backend Specialist Agent** — a Python-based AI agent that generates production-ready .NET 9 CQRS microservices. The agent takes a `WorkPackage` from the Orchestrator and produces complete, compilable C# code following all AD Ports coding standards.

The agent uses **Scriban templates** (not raw LLM generation) for boilerplate and **LLM** only for business logic interpretation. This hybrid approach ensures 90%+ build success on first attempt.

## Template Engine Pattern (Scriban)

```python
# agents/backend_agent/templates/command_handler.scriban.cs
// {{ entity }}CommandHandler.cs — AUTO-GENERATED by AD Ports Backend Agent
// DO NOT EDIT manually. Re-run via Portal to regenerate.

using MediatR;
using FluentResults;
using {{ namespace }}.Domain.{{ bounded_context }};

namespace {{ namespace }}.Application.{{ bounded_context }}.Commands;

public class {{ verb }}{{ entity }}CommandHandler
    : IRequestHandler<{{ verb }}{{ entity }}Command, Result<{{ return_type }}>
>
{
    private readonly I{{ entity }}DbContext _context;
    private readonly ILedgerClient _ledger;
    private readonly ILogger<{{ verb }}{{ entity }}CommandHandler> _logger;

    public {{ verb }}{{ entity }}CommandHandler(
        I{{ entity }}DbContext context,
        ILedgerClient ledger,
        ILogger<{{ verb }}{{ entity }}CommandHandler> logger)
    {
        _context = context;
        _ledger  = ledger;
        _logger  = logger;
    }

    public async Task<Result<{{ return_type }}>> Handle(
        {{ verb }}{{ entity }}Command request,
        CancellationToken cancellationToken)
    {
        {{ business_logic }}

        await _context.SaveChangesAsync(cancellationToken);

        await _ledger.RecordAsync(new LedgerEvent
        {
            EventType = "{{ domain | downcase }}.{{ entity | downcase }}.{{ past_tense_verb | downcase }}",
            ActorId   = request.CurrentUserId,
            EventData = new { /* relevant fields */ },
        }, cancellationToken);

        return Result.Ok({{ return_value }});
    }
}
```

## LLM Usage in Backend Agent

The LLM is used ONLY for:
1. **Business logic interpretation** — translating acceptance criteria to C# business rules
2. **Entity property identification** — extracting domain entities from requirements
3. **Validator rules** — deriving FluentValidation rules from acceptance criteria

The LLM is NOT used for:
- Handler boilerplate (use Scriban template)
- EF Core configuration (use template)
- Migration generation (use `dotnet ef migrations add`)
- OpenAPI annotations (use template)

```python
# agents/backend_agent/nodes/business_logic_node.py

BUSINESS_LOGIC_PROMPT = """
You are generating C# business logic for an AD Ports .NET CQRS handler.

Given these acceptance criteria:
{acceptance_criteria}

Generate ONLY the business logic body (the code that goes inside the Handle method, 
before the SaveChangesAsync call). 

Rules:
- Use domain factory methods: {Entity}.Create(...) not new {Entity}(...)
- Validate using domain invariants, not FluentValidation (that's in the validator class)
- Return Result.Failure<T>(Error.Validation(...)) for business rule violations
- Do NOT include SaveChangesAsync or Ledger recording — those are in the template

Output: A C# code block only. No explanation.
"""
```

## Build Verification Step

After generating all files, the agent MUST run a compilation check:

```python
async def verify_compilation(output_dir: str) -> BuildResult:
    """Run dotnet build to verify generated code compiles."""
    result = await asyncio.create_subprocess_exec(
        "dotnet", "build", output_dir,
        "--configuration", "Debug",
        "--verbosity", "minimal",
        "--nologo",
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE,
    )
    stdout, stderr = await result.communicate()
    
    if result.returncode != 0:
        return BuildResult(
            success=False,
            errors=parse_build_errors(stderr.decode()),
            output=stdout.decode(),
        )
    return BuildResult(success=True)
```
